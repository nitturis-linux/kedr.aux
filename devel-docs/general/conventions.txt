    Соглашения, принятые в рамках проекта KEDR.

Упорядоченности нет, соглащения пишутся по мере обсуждения/вспоминания обсужденного.

1.Соглашения по путям установки файлов вынесены в paths_convensions.txt.

2. Включаемые файлы для пространств пользователя и ядра.

Включаемый файлы для внутренних нужд сборки располагаются, где удобнее.
Далее речь идет про включаемые файлы, поставляемые вместе с пакетом.

Вне зависимости от того, для какого пространства создан включаемый файл, располагатся они будут в одних директориях(см. соглашения по путям установки файлов, "включаемые файлы").

a)Если файл предназначен для включения только для пространства ядра,
то в нем должна присутствовать защита всех kernel данных через "ifdef __KERNEL__".

b)Если файл предназначен для включения как в пространстве ядра, так и в пространстве пользователя, то все "только ядерные данные" должны быть защищены через "ifdef __KERNEL__", все "только пользовательские данные" должны быть защищены "ifndef __KERNEL__" или аналогичным способом.

c) Если файл предназначен для включения только в пространстве пользователя, защиты данных можно не использовать. Разве что, на случай, что файл может быть спутан с другим, предназначенным для включения в пространстве ядра.

3. Все включаемые файлы, предназначенные для установки, должны лежать в поддиректориях ${src}/include/kedr/, и воссоздавать дерево установленных файлов.

Иными словами, включаемый файл
    ${src}/include/kedr/$rel_path/$filename
при установке попадает в
    ${install_include_dir}/$rel_path/$filename,
где
${install_include_dir} - директория для установки включаемых файлов(см. соглашения по путям установки файлов, "включаемые файлы")

4. Для использования включаемых файлов в коде следует использовать их относительный путь после установки, включая ведущую директорию "kedr":
#include <kedr/$rel_path/$filename>

Использование вспомогательных включаемых файлов не регламентируется.
