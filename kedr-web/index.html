About KEDR

KEDR framework is an extensible system to facilitate runtime analysis of kernel modules in Linux. It allows to intercept the calls that a kernel module makes to the functions exported by other modules and by the kernel proper. KEDR-based systems can record the arguments and return values of these functions to a trace, do fault simulation according to user-defined scenarios and perform many other tasks. KEDR framework can be a valuable asset in revealing errors in the kernel modules, for example.

More Info

Detailed information about KEDR can be found in the online documentation. The documentation is also distributed along with the source code of the framework. In addition, Wiki provides other useful information concerning KEDR.

To begin with KEDR, you can take a look at Getting Started section in the manual. It describes how to install the framework and use it for call monitoring and fault simulation, step by step.

Checking Kernel Modules for Memory Leaks

One of the important applications of KEDR framework is checking kernel modules for memory leaks. Such errors in the modules can be really nasty and very hard to find.

There is a special tool based on KEDR framework to reveal memory leaks, "kedr_leak_check", that usually incurs no significant overhead in memory and performance. "kedr_leak_check" is easy to use, it does not require complex installation or configuration procedure. It just works out-of-the-box. Simply install KEDR and build "kedr_leak_check". Choose which kernel module you would like to analyze and then load the KEDR core and the tool and let them do the job. See Readme.txt included in the archive with "kedr_leak_check" for details.

"kedr_leak_check" is likely to be included into KEDR framework in the future releases. For now, it is distributed separately (see "Downloads" section below).

Mercurial Repository

The latest version of KEDR sources can be found in the repository at the project site. The repository is maintained with Mercurial source control management system.

If you would like to get a local copy of the repository, use the following command: 

hg clone http://hg.berlios.de/repos/kedr kedr-local
