About KEDR

KEDR framework is an extensible system to facilitate runtime analysis of kernel modules in Linux. It allows to intercept the calls that a kernel module makes to the functions exported by other modules and by the kernel proper. KEDR-based systems can record the arguments and return values of these functions to a trace, do fault simulation according to user-defined scenarios and perform many other tasks. Such systems can be valuable assets in revealing the errors in the kernel modules, for example.

More Info

Detailed information about KEDR can be found in the online documentation. The documentation is also distributed along with the source code of the framework. In addition, Wiki provides other useful information concerning KEDR.

To begin with KEDR, you can take a look at Getting Started section in the manual. It describes how to install the framework and use it for call monitoring and fault simulation, step by step.

Downloads

KEDR framework is distributed in the source form. The current stable version is 0.1 (download).

The source archive also includes the documentation (in DocBook and HTML formats) and the examples.

Mercurial Repository

The latest version of KEDR sources can be found in the repository at the project site. The repository is maintained with Mercurial source control management system.

If you would like to get a local copy of the repository, use the following command: 

hg clone http://hg.berlios.de/repos/kedr kedr-local
