About KEDR

KEDR framework is an extensible system for dynamic analysis of kernel modules (device drivers, file system modules, etc.) in Linux. KEDR tools operate on the modules chosen by the user and can detect memory leaks, perform fault simulation and more. The framework is intended for the developers of kernel modules and, in particular, it may be useful for building automated driver verification systems.

Features

Like the dynamic analysis frameworks, such as Valgrind, oriented towards user-space applications and libraries, KEDR uses plugin-based architecture. Each analysis system based on KEDR relies at least on the following components:

  • KEDR Core which is responsible for instrumentation of the module being analyzed;
  • One or more tool plugins ("payload modules") that actually collect and analyze the data.

Out-of-the-box, KEDR provides the tools to do the following:

  • Call monitoring (call tracing) During the operation of the module under analysis, the information is collected about the function calls it makes: arguments, return values, etc. The data can be saved to a file (trace) for future analysis. This is similar to what strace utility does for user-space applications.

  • Fault simulation KEDR forces some of the calls made by the target module to fail. In fact, KEDR simulates a failure without actually calling the respective function. The scenarios (the calls to which functions must fail in what conditions) can be controlled and customized by the user.

  • Detecting memory leaks KEDR keeps track of memory allocation and deallocation operations made by the target module. After the target module has unloaded, KEDR generates a report listing the memory blocks that have been allocated but not freed by that module along with a call stack for each of these blocks.

Other types of data collection and analysis operations can also be implemented.

Downloads

Mercurial Repository

The latest version of KEDR sources can be found in the repository at the project site. The repository is maintained with Mercurial source control management system.

If you would like to get a local copy of the repository, use the following command: 

hg clone http://hg.berlios.de/repos/kedr kedr-local

Bugs and Features

If you have found a problem in KEDR, please report it via the bug tracking system. Another tracking system is used to manage feature requests and enhancement proposals.

  • kedr-users - discussion of the usage of KEDR.
